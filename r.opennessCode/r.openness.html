NAME

r.openness - Calculate different approaches to the openness parameter (Yokoyama et al., 2002)

KEYWORDS

raster, focal, geomorphometry

SYNOPSIS

r.openness
r.openness help
r.openness elev=name flowdir=name popenness8=name popennesstot=name popennessup=name popennessdn=name popennessbn=name nopenness8=name nopennesstot=name nopennessup=name nopennessdn=name nopennessbn=name size=value mode=value

Flags:

--overwrite
Allow output files to overwrite existing files
--verbose
Verbose module output
--quiet
Quiet module output

Parameters:

elev=name
Name of the MDE raster map
flowdir=name
Name of a second input raster map with the flow directions needed to calcula openness in different directions
popenness8=name
Positive Openness calculated with the 8 main directions
popennesstot=name
Positive Openness calculated with all directions
popennessup=name
Positive Openness calculated in the upslope direction
popennessdn=name
Positive Openness calculated in the downslope direction
popennessbn=name
Positive Openness calculated in the direction transverse to the flow
nopenness8=name
Negative Openness calculated with the 8 main directions
nopennesstot=name
Negative Openness calculated with all directions
nopennessup=name
Negative Openness calculated in the upslope direction
nopennessdn=name
Negative Openness calculated in the downslope direction
nopennessbn=name
Negative Openness calculated in the direction transverse to the flow
size=integer
window size
mode=integer
window shape

DESCRIPTION

Yokoyama et al. (2002) presented a multi-scale geomorphometric feature called openness, as the average of the zenith (positive openness) or nadir (negative openness) angles along the main eight compass directions (N, NE, E, SE, S, SW, W, NW). Jasiewicz and Stepinski (2013) used this feature, with a slight modification when calculating negative openness. According to the original authors, openness expresses a topographic attribute possibly not measured by other features.

The approach in this module is different as all the directions available for a given window size (w) are used to calculate the avarages; the nuber of directions available is 4(w-1). However, the module also allows to calculate openness with the original 8 directios approach. In addition, this module can calculate both positive and negative openness in the downslope, upslope and transverse to the flow directions. To do so, the user should provide a map of flow directions.

The user can select which types of output maps will be produced, the window size and the window shape: 1 (default) for square and 2 for circle. If mode=2 size is the diameter of the circle.

The details of the algorithms are described in a paper submited to Computers & Geosciences.
